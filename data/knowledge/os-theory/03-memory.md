# 内存管理

## 地址空间

### 物理地址与虚拟地址

- **物理地址**：实际的内存硬件地址
- **虚拟地址**：程序使用的逻辑地址
- **地址转换**：通过 MMU（内存管理单元）将虚拟地址转换为物理地址

### 地址空间的隔离

每个进程拥有独立的虚拟地址空间，实现：
- **隔离**：进程间互不干扰
- **保护**：防止非法访问
- **共享**：必要的共享内存区域

## 内存分配策略

### 连续内存分配

1. **固定分区**：内存划分为固定大小的区域
2. **动态分区**：根据进程需求动态分配
3. **伙伴系统**：将内存按 2 的幂次分割

### 分页机制

**分页（Paging）**将虚拟内存和物理内存划分为固定大小的页：

- **页（Page）**：虚拟内存的单位
- **页框（Frame）**：物理内存的单位
- **页表（Page Table）**：记录虚拟页到物理页框的映射

```
虚拟地址 = [虚拟页号 | 页内偏移]
物理地址 = [物理页框号 | 页内偏移]
```

### 多级页表

为了减少页表占用的内存：
- **一级页表**：直接映射
- **二级页表**：页目录 + 页表
- **多级页表**：进一步分层

## 虚拟内存

### 按需分页（Demand Paging）

- 页面在需要时才加载到内存
- 减少内存占用
- 支持更大的地址空间

### 页面置换算法

当内存不足时，需要选择一个页面换出：

1. **FIFO（先进先出）**：最早进入的页面先换出
2. **LRU（最近最少使用）**：最久未使用的页面换出
3. **OPT（最优）**：换出将来最久不会使用的页面（理论最优）
4. **Clock 算法**：LRU 的近似实现

### 页面置换的性能

- **缺页率**：访问页面时需要从磁盘加载的比例
- **有效访问时间**：EAT = (1-p) × 内存访问时间 + p × 缺页处理时间

## 内存保护

- **界限寄存器**：检查地址是否在合法范围内
- **页表权限位**：读/写/执行权限
- **TLB（快表）**：加速地址转换的缓存
