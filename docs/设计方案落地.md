## 一、架构落地

### 1.1 系统架构

```
用户浏览器
    ↓
Next.js API Routes
    ↓
Router Agent → Judge 模块 → 专业 Agent
    ↓
记忆系统 (L1/L2/L3)
    ↓
数据库 + 知识库
```

| 层级 | 选型 | 版本 |

| 数据库 | SQLite → PostgreSQL | - |

### 2.2 Judge 模块

**文件**: `lib/judge/index.ts`

**职责**: 判断学员是否已尝试过某道题

**规则引擎**:
- R1: 学员提交了代码 → 已尝试
- R2: 学员运行了测试 → 已尝试
- R3: 学员点击了"我尝试了"按钮 → 已尝试
- R4: 学员说"我试过了" → 已尝试
- R5: 学员刚看到题目 (< 30秒) → 未尝试
- R6: 无交互记录 → 未尝试
- F1-F4: 模糊情况 → LLM 判断

## 三、数据落地

### 3.1 数据库 Schema

**文件**: `prisma/schema.prisma`

**表结构**:
- `Student` - 学员档案
- `Assessment` - 水平评估
- `Question` - 题目
- `AnswerRecord` - 答题记录
- `LearningPlan` - 学习计划

### 3.2 数据保留策略

- 60 天无记录 → 自动删除详细对话历史
- 保留：学习进度、知识点掌握度、最终成绩
- 个人上限：对话 1000 条，代码 100 次
