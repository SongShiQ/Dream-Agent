# 项目文档与目标

**项目名称**: Dream Agent (OpenCamp AI 助教)
**日期**: 2026-07-11
**版本**: v1.0

---

## 一、项目概述

### 1.1 背景

OpenCamp 是清华大学陈渝、向勇老师发起的开源操作系统训练营，2026 春夏季有 2400+ 学员来自 723 所高校。

### 1.2 核心问题

- 老师精力有限，无法答疑所有学员
- 学员水平参差不齐，需要个性化指导
- AI 已经可以胜任助教角色

### 1.3 项目目标

构建一个 **AI 助教系统**，能够：
1. 替代训练营助教的部分工作
2. 根据学员水平提供个性化指导
3. 后续接入训练营主页，作为官方接口

---

## 二、产品目标

### 2.1 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **智能答疑** | 回答 OS 相关问题，引导式教学 | P0 |
| **出题练习** | 根据学员水平动态出题 | P0 |
| **水平评估** | 评估学员当前阶段 | P0 |
| **学习规划** | 制定个性化学习路径 | P1 |
| **进度跟踪** | 记录学习进度和知识点掌握度 | P1 |

### 2.2 用户流程

```
用户访问 → 无需登录 → 直接开始聊天 → AI 问问题 → 水平自测 → 确定阶段 → 开始学习
```

### 2.3 难度划分

| 阶段 | 可跳过 | 细分 | 说明 |
|------|--------|------|------|
| 导学 | ✅ | A1-A3 | A1 零基础、A2 有编程经验、A3 有其他语言基础 |
| 基础 | ✅ | B1-B3 | B1 Rust 入门、B2 Rust 进阶、B3 工具使用 |
| 专业 | ❌ | C1-C5 | rCore 各模块（批处理、地址空间、进程、文件系统、并发） |
| 项目 | 需完成专业 | D1-D2 | D1 组件化 OS、D2 项目实践 |

## 六、风险与约束

### 6.1 风险

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM 幻觉 | 回答不准确 | RAG + 引用来源 |
