# 本地优先教学 Agent 目标态

**日期**：2026-07-16  
**状态**：已冻结为产品方向  
**前提**：不追求在线大规模 OJ；本地能跑、能教、能练、能反馈即可。怎么方便怎么来。

## 1. 我们要做成什么

做一个**属于 OpenCamp 的教学 Agent**，效果对标并融合：

| 参考 | 我们吸收的“效果” | 我们不照搬的 |
|---|---|---|
| DeepTutor | 导学、测验、记忆、掌握度、多 Agent 分工 | Python 全栈重写 |
| OpenKB / LLM Wiki | 课程知识可编译、可引用、可维护 | 一上来上重型独立知识服务 |
| llm-wiki-skills | Agent 维护知识的操作规程 | 仅停留在 skill 文档 |
| 现有 Dream Agent | 训练营路径、题库、Lab 关卡、本地提交 | 在线判题平台包袱 |

一句话：

> **本地可运行的 OS 教学 Agent = Wiki 知识层 + Tutor 教学闭环 + 本地练习/实验反馈。**

## 2. 本地优先原则

1. **单机可启动**：`npm run dev` + 本地 SQLite 即可完整体验。  
2. **判题可本地**：进程内 / 本机 `cargo test` / 可选 Docker；不依赖公网 OJ。  
3. **知识可落地文件**：Markdown 课程 Wiki / 知识卡，Git 可审可回滚。  
4. **LLM 可插拔**：无 Key 时降级为检索 + 题库 + 规则路径；有 Key 时增强讲解与出题。  
5. **过关证据本地可信**：浏览 ≠ 完成 ≠ 掌握；本地 AC / 小测达标才算会。  
6. **先纵向切片，再铺全课**：先做一个主题闭环打穿，再扩章节。

## 3. 效果目标（验收用）

学生打开本地系统后应稳定得到：

1. 我在哪、下一步只做什么  
2. 讲解能回到课程条目（有出处）  
3. 薄弱点能变成题组 / 微单元 / 小实验  
4. 本地跑通或小测达标才算前进  
5. 高频坑位能沉淀回公共知识，而不是只留在聊天里

## 4. 模块分工（在 Dream Agent 内）

```text
data/knowledge + data/curriculum   → 课程 Wiki / 内容包（类 OpenKB 产物）
lib/agents/*                       → 教学运行时（类 DeepTutor）
lib/knowledge/*                    → 检索与引用
lib/labs/* + lib/judge/*           → 本地练习与实验反馈
prisma + lib/progress/*            → 学习证据与掌握度
Web UI + CLI                       → 学生入口
```

## 5. 近期怎么推进（方便优先）

### P0 本周可做
- 冻结本方向文档与调研结论（本次）  
- 收干净现有 funnel / foundation / local judge 脚手架并提交  
- 选定 1 个主题切片（建议：Rust 所有权 或 进程/系统调用）

### P1 下一个切片
- 把该主题 knowledge 做成互链 Wiki 页  
- Tutor 强制引用条目 ID  
- 本地小测 + 1 个可本机验证的 mini lab/gate

### P2 再增强
- 失败模式回写 Wiki（需轻量审核标记）  
- 掌握度面板与“下一步”统一  
- 可选 Docker 判题，仍然本地

## 6. 明确不做

- 不做“必须在线才能学”的平台包袱  
- 不把 LLM 主观分当成过关  
- 不并行重写 DeepTutor / OpenKB 全套  
- 不为 2000 人在线容量过早优化

## 7. 成功标准

当我们愿意对外说“这是我们的教学 Agent”时，至少满足：

- 本地 10 分钟内可跑通：诊断 → 学一页 → 做几题 → 做一个小关 → 看到进度  
- 关键回答带知识出处  
- 至少一个 OS/Rust 主题形成 Wiki→教→练→证闭环  
- 代码与内容都在本仓库，可 Git 管理
