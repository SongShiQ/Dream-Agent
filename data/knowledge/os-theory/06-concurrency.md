# 并发与同步

## 并发的概念

**并发（Concurrency）**是指多个任务在同一时间段内交替执行，与**并行（Parallelism）**（同时执行）不同。

### 并发的挑战

1. **竞态条件（Race Condition）**：多个线程访问共享数据，结果取决于执行顺序
2. **数据不一致**：并发修改导致数据状态错误
3. **死锁（Deadlock）**：多个线程互相等待，都无法继续

## 临界区问题

**临界区（Critical Section）**是访问共享资源的代码段，需要互斥执行。

### 临界区解决方案的要求

1. **互斥（Mutual Exclusion）**：同一时刻只有一个线程进入临界区
2. **前进（Progress）**：如果没有线程在临界区，等待的线程可以进入
3. **有限等待（Bounded Waiting）**：线程等待进入临界区的时间是有限的

## 同步机制

### 锁（Lock）

```rust
// 互斥锁
let mutex = Mutex::new(data);
{
    let mut guard = mutex.lock().unwrap();
    *guard += 1;
} // 锁在这里释放
```

### 信号量（Semaphore）

信号量是一个整数计数器，支持两个原子操作：
- **P（wait）**：如果计数 > 0，减 1 并继续；否则等待
- **V（signal）**：加 1，唤醒等待的线程

```rust
// 二进制信号量（互斥锁）
let semaphore = Semaphore::new(1);

// 计数信号量（资源池）
let semaphore = Semaphore::new(10);
```

### 管程（Monitor）

管程是一种高级同步原语，封装了共享数据和操作：
- 同一时刻只有一个线程可以执行管程内的方法
- 支持条件变量（condition variables）

## 死锁

### 死锁的四个必要条件

1. **互斥**：资源不能共享
2. **持有并等待**：线程持有资源的同时等待其他资源
3. **不可抢占**：资源只能由持有者释放
4. **循环等待**：存在线程的循环等待链

### 死锁处理策略

1. **预防**：破坏四个条件之一
2. **避免**：使用银行家算法等动态检查
3. **检测与恢复**：允许死锁发生，检测后恢复
4. **忽略**：鸵鸟策略（大多数操作系统采用）

### 死锁预防

- **一次性申请**：破坏"持有并等待"
- **资源排序**：破坏"循环等待"
- **可抢占**：破坏"不可抢占"

## 经典同步问题

### 生产者-消费者问题

```
生产者 → 缓冲区 → 消费者
```

需要：
- 互斥访问缓冲区
- 缓冲区满时生产者等待
- 缓冲区空时消费者等待

### 读者-写者问题

```
多个读者可以同时读
写者必须独占访问
```

策略：
- 读者优先：读者可以持续读
- 写者优先：写者等待时阻止新读者

### 哲学家就餐问题

```
5 个哲学家，5 根筷子
每人需要 2 根筷子才能吃饭
```

解决方案：
- 最多允许 4 人同时尝试拿筷子
- 奇数哲学家先拿左边，偶数先拿右边
