# 文件系统

## 文件的概念

**文件**是存储在持久化存储设备上的数据集合，是操作系统管理数据的基本单位。

### 文件属性

- **名称**：人类可读的标识符
- **类型**：普通文件、目录、设备文件等
- **大小**：文件占用的存储空间
- **位置**：文件在存储设备上的地址
- **保护**：访问权限（读/写/执行）
- **时间**：创建、修改、访问时间

### 文件操作

- **create**：创建新文件
- **delete**：删除文件
- **open**：打开文件获取文件描述符
- **close**：关闭文件释放资源
- **read**：从文件读取数据
- **write**：向文件写入数据
- **seek**：移动文件指针位置

## 目录结构

### 目录组织方式

1. **单级目录**：所有文件在同一目录
2. **两级目录**：每个用户有自己的目录
3. **树形目录**：层次化的目录结构
4. **无环图目录**：支持文件共享

### 路径

- **绝对路径**：从根目录开始的完整路径
- **相对路径**：从当前目录开始的路径

## 文件系统实现

### 磁盘分配方法

1. **连续分配**：文件占用连续的磁盘块
   - 优点：读取速度快
   - 缺点：外部碎片、文件大小受限

2. **链接分配**：文件块通过指针链接
   - 优点：无外部碎片
   - 缺点：随机访问慢、指针开销

3. **索引分配**：使用索引块存储块地址
   - 优点：支持随机访问
   - 缺点：索引块开销

### inode 结构

Unix 文件系统使用 **inode** 存储文件元数据：

```
inode 包含：
- 文件类型和权限
- 所有者信息
- 文件大小
- 时间戳
- 数据块指针（直接/间接）
```

### 日志文件系统

为了防止系统崩溃导致数据不一致：
1. **写日志**：先将操作记录到日志
2. **执行操作**：实际修改文件系统
3. **清除日志**：操作完成后清除日志

## 文件系统实例

### ext4

- Linux 默认文件系统
- 支持大文件和大容量
- 日志文件系统

### FAT

- 简单的文件系统
- 广泛用于 USB 设备
- 不支持权限控制
