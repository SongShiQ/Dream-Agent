# 输入输出管理

## I/O 硬件原理

### I/O 设备分类

1. **块设备**：以固定大小块为单位传输数据
   - 磁盘、SSD
   - 支持随机访问

2. **字符设备**：以字符流形式传输数据
   - 键盘、鼠标、串口
   - 顺序访问

### I/O 控制方式

1. **轮询（Polling）**：CPU 主动检查设备状态
   - 简单但浪费 CPU

2. **中断（Interrupt）**：设备完成操作后通知 CPU
   - CPU 可以做其他工作
   - 需要中断处理程序

3. **DMA（Direct Memory Access）**：设备直接访问内存
   - 减少 CPU 负担
   - 适合大数据传输

## I/O 软件层次

```
用户程序
    ↓
设备无关软件
    ↓
设备驱动程序
    ↓
中断处理程序
    ↓
硬件
```

### 各层职责

1. **用户程序**：发起 I/O 请求
2. **设备无关软件**：提供统一接口，缓冲管理
3. **设备驱动程序**：控制特定设备
4. **中断处理程序**：响应设备中断
5. **硬件**：实际执行 I/O 操作

## 磁盘调度算法

磁盘访问的主要开销是**寻道时间**，调度算法的目标是减少磁头移动：

1. **FCFS（先来先服务）**：按请求顺序执行
2. **SSTF（最短寻道时间优先）**：选择最近的请求
3. **SCAN（电梯算法）**：磁头来回扫描
4. **C-SCAN（循环扫描）**：单向扫描
5. **LOOK**：到达最远请求后折返

## 缓冲技术

### 缓冲的作用

1. **解决速度不匹配**：CPU 与 I/O 设备速度差异
2. **减少中断次数**：批量处理数据
3. **提高并行性**：CPU 和 I/O 可以并行工作

### 缓冲类型

1. **单缓冲**：一个缓冲区
2. **双缓冲**：两个缓冲区交替使用
3. **循环缓冲**：多个缓冲区循环使用
4. **缓冲池**：共享的缓冲区集合
