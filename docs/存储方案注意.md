# 存储方案注意文档

**项目名称**: Dream Agent (OpenCamp AI 助教)
**日期**: 2026-07-11
**版本**: v1.0

---

## 一、存储架构

### 1.1 数据分类

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 代码提交 | Git 仓库 + 数据库 | Git 存代码，数据库存元数据 |
| 学习进度 | 数据库 | 需要快速查询 |
| 对话历史 | 数据库 | 需要检索和分析 |
| 知识库 | 向量数据库 | 需要语义搜索 |

### 1.2 数据库选型

| 阶段 | 数据库 | 理由 |
|------|--------|------|
| 开发 | SQLite | 零配置，快速启动 |
| 生产 | PostgreSQL | 功能完整，支持向量搜索 |

---

## 二、数据保留策略

### 2.1 自动删除规则

- **60 天无新记录** → 自动删除详细对话历史
- **保留**：学习进度、知识点掌握度、最终成绩
- **删除**：详细对话历史、中间代码版本

### 2.2 个人数据上限

| 数据类型 | 上限 | 理由 |
|----------|------|------|
| 对话历史 | 1000 条 | 足够回溯学习过程 |
| 代码提交 | 100 次 | 足够覆盖所有 lab |
| 学习进度 | 无限制 | 数据量小 |

### 2.3 服务器压力预估

- 每个学员：~10MB（压缩后）
- 2400 学员：~24GB（完全可控）
- 即使翻倍也是 48GB（现代服务器轻松应对）

---

## 三、非 OS 内容处理

### 3.1 策略

**非 OS 内容不归档到数据库**

### 3.2 实现方式

在 Router 层判断：
```
学员提问 → Router 判断意图
  ├─ OS 相关 → 存入数据库
  └─ 非 OS 相关 → 不存入数据库，只回复不记录
```

### 3.3 判断依据

- 关键词匹配（OS、进程、内存、文件系统等）
- LLM 判断（更准确）

---

## 四、数据备份

### 4.1 备份策略

- **每日备份**：数据库完整备份
- **每周备份**：Git 仓库完整备份
- **每月备份**：全量备份到云存储

### 4.2 恢复策略

- **数据丢失**：从最近备份恢复
- **误删除**：从回收站恢复（保留 30 天）

---

## 五、安全注意事项

### 5.1 数据安全

- **加密存储**：敏感数据加密存储
- **访问控制**：严格的权限管理
- **审计日志**：记录所有数据访问

### 5.2 隐私保护

- **学员数据**：仅用于学习分析
- **对话记录**：匿名化处理
- **代码提交**：仅用于 AI 分析
