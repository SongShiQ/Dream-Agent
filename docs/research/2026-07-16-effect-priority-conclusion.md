# DeepTutor × LLM Wiki × OpenKB × OS 实验框架：效果优先结论

**日期**：2026-07-16  
**性质**：不论工程难度，只比较最终学习效果；并给出 OpenCamp / Dream Agent 应追求的效果上限。  
**关联调研**：[2026-07-16-deeptutor-llm-wiki-openkb-os-lab-research.md](./2026-07-16-deeptutor-llm-wiki-openkb-os-lab-research.md)

**源码复核**：[2026-07-17-four-repos-source-audit.md](./2026-07-17-four-repos-source-audit.md)

## 1. 一句话结论

**最终效果最好的，不是单押 DeepTutor，也不是单押 LLM Wiki，而是：**

> **结构化课程 Wiki（知识可编译、可引用、可维护）  
> \+ DeepTutor 式教学运行时（诊断 / 路径 / 记忆 / 掌握度）  
> \+ 真实 OS 实验与 OJ 证据（会做、可复现、能过关）**

对 OpenCamp 这种“2000 人导学分流 → 少数人进项目”的 OS 课，**三层闭环的效果上限最高**；任何一层单独做，都会在完课率、概念迁移或可信晋级上明显掉档。

## 2. 效果维度（比什么）

| 维度 | 含义 | 为何重要 |
|---|---|---|
| 概念理解深度 | 能否解释机制、对比方案、定位根因 | OS 概念高度耦合 |
| 知识稳定性 | 前后是否一致、来源是否可追溯 | 避免“今天一个说法、明天另一个” |
| 个性化陪伴 | 是否按薄弱点推进，而不是统一灌输 | 导学层人数大 |
| 学习驱动力 | 是否知道下一步、是否持续回来 | 决定真实完课 |
| 动手迁移 | 概念能否变成能 AC 的代码/实验 | 训练营核心产出 |
| 晋级可信度 | 过关是否可复测、不可刷 | 倒金字塔分流生命线 |
| 长期资产 | 课程知识是否越用越厚 | 多届复用、助教减负 |

## 3. 四种路线的最终效果对比

### A. 只有 LLM Wiki / OpenKB（强知识，弱教学）

**擅长**：把讲义、文档、实验说明编译成互相链接的知识网；答案有出处；可 lint 掉矛盾与缺口。

**学生侧最终观感**：高级课程资料站 + 可检索第二大脑。  
**强项**：知识质量、可审计、可积累。  
**弱项**：缺诊断与路径、缺掌握度压迫感、缺“下一步只做什么”；自律强的学生受益大，大多数人容易停在“看懂了但没练完”。

**效果上限**：中高（适合尖子生自学与教研资产）；**对 2000 人导学不够**。

### B. 只有 DeepTutor 式教学中枢（强教学，弱知识编译）

**擅长**：导学、多轮讲解、测验、记忆、掌握度推进，学生“被带着学”。

**学生侧最终观感**：很会教的 AI 家教。  
**强项**：陪伴感、路径感、完课推动力。  
**弱项**：若底层仍是临时检索/散卡片，OS 高耦合知识点容易漂、难统一口径；讲解爽，但深度与一致性 dev 上限受限。

**效果上限**：中高（短期体验与坚持度好）；**长期课程资产与口径治理偏弱**。

### C. 只有实验 / OJ / 实验生成（强动手，弱教与知）

**擅长**：真跑测、AC 纪律、任务驱动。

**学生侧最终观感**：严格 OJ + 实验平台。  
**强项**：动手能力、晋级可信。  
**弱项**：卡住时系统性补概念弱；容易“会过测试但说不清为什么”；弱基础学生流失高。

**效果上限**：中（选拔有效，培养完整度不足）。

### D. Wiki + Tutor + Lab/OJ 三层合体（推荐效果目标）

**学生最终得到的不是聊天，而是：**

1. 想查：有结构化、可引用的课程 Wiki  
2. 想学：有诊断、补弱路径、小测与记忆  
3. 想练：有关卡实验与真实 AC  
4. 卡住：能从失败证据回到概念页/微单元，而不是盲刷  
5. 晋级：靠证据，不靠自报或静态分

**效果上限：最高。**  
尤其匹配 OS 课：“概念网状 + 必须动手 + 必须可信分流”。

## 4. 效果排名（忽略实现成本）

| 排名 | 路线 | 综合学习效果 | 最适场景 |
|---:|---|---|---|
| 1 | **Wiki + DeepTutor 闭环 + 真实验 OJ** | 最高 | OpenCamp 目标态 |
| 2 | DeepTutor 闭环 + 真实验 OJ（知识层较薄） | 高 | 先把学生“学完练过”，知识治理后补 |
| 3 | Wiki + 真实验 OJ（教学编排弱） | 中高 | 强自驱小班 / 教研资产优先 |
| 4 | 仅 DeepTutor 式陪伴答疑 | 中 | 辅导班体验，缺硬核验收 |
| 5 | 仅 LLM Wiki / OpenKB | 中 | 知识库产品，不是训练营 |
| 6 | 仅聊天式 RAG 答疑 | 低 | 演示友好，培养效果最差 |

补充：

- 若问“**学生每天打开最有感觉**”：Tutor 闭环 > Wiki  
- 若问“**三年后课程是否越来越强**”：Wiki 治理 > 纯 Tutor  
- 若问“**能不能信晋级结果**”：真 OJ / 实验证据 > 一切 LLM 自评  
- 若问“**OpenCamp 最终应追求什么**”：三者缺一，效果都会在某一维塌方

## 5. 对“现有 Dream Agent 上增强”的效果判断

不论好不好做，只谈做成后的效果：

| 目标态 | 相对现状的效果跃迁 |
|---|---|
| 把 knowledge cards 升级为可链接、可引用、可 lint 的课程 Wiki | 讲解一致性、助教复用、错题归因显著变好 |
| 吸收 DeepTutor 的掌握度/记忆/统一学生状态（不一定搬代码） | “下一步做什么”更清楚，导学完课率上升 |
| 吃透 unit/integration OJ 与实验生成 | 训练营区分度与项目选拔可信度上升 |
| 三者打通：弱项 → Wiki 概念 → 微单元/题目 → Lab AC → 回写 | **这才是自学系统，而不是功能拼盘** |

因此：  
**效果最好的目标态 = 合体**；  
**现有框架最值得保留的效果资产 = 已有学习闭环 + 题库 + 关卡/OJ 纪律**；  
**最值得补的效果短板 = 课程知识编译与引用治理（Wiki 层）**。

## 6. 一个可感知的“终局体验”标准

系统做成后，一个学生应稳定获得这 5 个体验（用来验收效果，不验收技术）：

1. **10 秒定位**：我在哪、下一步只做什么、怎样算过、卡住去哪。  
2. **有出处的懂**：每个关键解释能回到课程条目，而不是模型临场发挥。  
3. **补弱可执行**：诊断出的漏洞能变成具体微单元/题组/关卡。  
4. **做过才算会**：Lab/OJ AC 是硬证据；AI 题与浏览不进 passed。  
5. **越学越厚**：高频坑位与优秀复盘经审核进入公共 Wiki，下届直接受益。

达不到 1–4，就还是工具；达到 1–5，才是可运营的 OS 自学系统。

## 7. 决策冻结（效果导向）

1. **效果目标冻结为三层合体**，不单押任一开源项目。  
2. DeepTutor：主要吸收**教学状态机与记忆/掌握度思想**。  
3. OpenKB / llm-wiki-skills / llm_wiki：主要吸收**知识编译、查询、维护协议**；OpenKB 更偏引擎，skills 更偏 Agent 工作流。  
4. Dream Agent：继续作为**学生入口、证据库、OJ 与运营壳**。  
5. 第一刀效果验证应用“单主题纵向切片”（建议：进程/地址空间 或 现有 foundation 某一维），而不是并行重写全站。

---

**最终判断**：不管实不实现得动，**最后效果最好的是三层合体**；若必须在学生可感知效果上砍一刀，优先保证 **Tutor 闭环 + 真 OJ**，Wiki 作为让该闭环“变深、变稳、可变资产”的关键倍增器，而不是可有可无的装饰。

## 9. 本地优先修订（2026-07-16）

评测层**不要求在线 OJ**。本地 `cargo test` / 进程判题 / 可选 Docker 均可。  
对教学效果而言，关键是“有可复现证据”，不是“有公网判题集群”。
