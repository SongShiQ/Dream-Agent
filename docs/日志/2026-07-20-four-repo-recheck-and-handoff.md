## 复核结论

- DeepTutor 是完整 Tutor 运行时；可独立 POC，最值得移植 mastery、policy 和 scheduler 思路，不作为 Dream Agent 的第二套后端。
- llm-wiki-skills 是知识维护操作规程，不是可执行知识引擎；直接采用其治理原则，不把它当服务部署。
- OpenKB 是教师/CI 侧离线知识编译器；适合直接承担多格式摄入、Wiki 编译和 lint，不进入学生在线请求热路径。
- nashsu/llm_wiki 是完整教师桌面知识产品；适合安装演示或通过导出/API/MCP 松耦合使用，不作为在线课程核心，也不直接复制 GPL-3.0 源码。
- 四个项目都不能替代 OS 实验生成、隔离执行、QEMU 判题和可信 AC 证据。

## Dream Agent 已落地能力核对

- 稳定知识条目 ID、Tutor 引用和聊天引用持久化。
- 确定性 mastery、下一任务策略、间隔复习和历史回填。
- OpenKB 单向导入、内容审计、发布门禁和教师审核队列。
- 教师内容决策、操作者身份、哈希防漂移、原子应用和审计历史。
- 受控 OS 形成性实验生成、学生尝试持久化和服务端评分。
- Wiki/Tutor/形成性实验不能授予正式 mastery；Lab Gate 仍要求受保护 Judge 返回 `AC`。
