# OS 训练营 AI 助教 - 多 Agent 架构设计文档

**日期**: 2026-07-11  
**状态**: 草稿 v2  
**方案**: 多 Agent 架构（用户驱动 LLM 选择）  
**训练营**: [OpenCamp 2026 春夏季开源操作系统训练营](https://opencamp.cn/os2edu/camp/2026spring)

---

## 1. 产品定位

### 一句话定义
一个能替代 OS 训练营老师大部分工作的多 Agent AI 系统，覆盖自适应出题、理论答疑、实验指导、进度跟踪，并支持用户自选 LLM。

### 核心用户
OpenCamp 训练营学员（2026 春夏季，2422人，723所高校）

### 核心价值
- **自适应出题**：根据学员水平动态生成个性化题目
- **7x24 即时答疑**：专业分工，避免上下文污染
- **个性化学习路径**：先评估水平，再制定方案，训练中动态调整
- **灵活 LLM 选择**：用户自己提供 API，框架不绑定特定模型

### Agent 角色设定
- **身份**：OpenCamp 训练营的 AI 助教团队
- **语气**：专业但友好，像一个耐心的 senior 学长
- **边界**：引导学员思考而非直接给答案

## 2. 多 Agent 架构

### 为什么用多 Agent？

| 问题 | 单 Agent 的问题 | 多 Agent 的解决方案 |
|------|----------------|-------------------|
| 上下文污染 | 出题逻辑混入答疑知识 | 专业 Agent 隔离上下文 |
| Prompt 膨胀 | 一个 prompt 塞所有功能 | 每个 Agent 专注自己的领域 |
| 动态调整 | 难以根据场景切换行为 | 调度 Agent 动态选择 |
| 可扩展性 | 加功能需重写整个 prompt | 新增 Agent 即可 |

### Agent 分工

### 各 Agent 详细职责

#### 调度 Agent (Router)
```
职责：分析用户意图，路由到合适的 Agent
输入：用户消息
输出：调用哪个 Agent + 参数

路由规则：
- 包含"出题"、"练习"、"测试" → 出题 Agent
- 包含"什么是"、"解释"、"为什么" → 答疑 Agent
- 包含"评估"、"我的水平" → 评估 Agent
- 包含"计划"、"怎么学" → 规划 Agent
- 复杂问题 → 多个 Agent 协作
```

#### 评估 Agent (Assessor)
```
职责：评估学员当前水平
输入：学员答题记录、对话历史
输出：水平评估报告

评估维度：
- 理论掌握程度 (0-100)
- 代码能力 (0-100)
- 当前阶段 (导学/基础/专业/项目)
- 薄弱知识点列表
```

#### 答疑 Agent (Tutor)
```
职责：回答 OS 相关问题
输入：学员问题 + 相关知识库检索结果
输出：解答 + 引用来源

工具：
- search_knowledge: 检索 OS 教材
- analyze_code: 分析 rCore/xv6 代码
```

#### 出题 Agent (Examiner)
```
职责：根据学员水平动态生成题目
输入：学员水平评估 + 当前学习阶段
输出：个性化题目

策略：
- 评估阶段：诊断性测试，快速定位水平
- 学习阶段：针对性练习，强化薄弱点
- 进阶阶段：挑战性题目，推动能力边界

题目类型：
- 选择题：概念理解
- 填空题：细节掌握
- 编程题：代码能力
- 设计题：系统思维
```

#### 规划 Agent (Planner)
```
职责：制定和调整学习计划
输入：学员水平 + 训练营大纲 + 答题表现
输出：个性化学习路径

计划内容：
- 当前阶段目标
- 每日/每周任务
- 推荐学习资源
- 晋级时间预估
```

### 难度调整算法

```typescript
// 简化的自适应算法
function adjustDifficulty(
  currentDifficulty: number,  // 0-100
  recentAccuracy: number,     // 最近10题正确率
  consecutiveCorrect: number  // 连续正确数
): number {
  if (recentAccuracy > 0.8 && consecutiveCorrect >= 3) {
    // 表现好，提升难度
    return Math.min(100, currentDifficulty + 10);
  } else if (recentAccuracy < 0.5) {
    // 表现差，降低难度
    return Math.max(0, currentDifficulty - 15);
  }
  return currentDifficulty;  // 保持不变
}
```

### 技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| **前端** | Next.js 14 + shadcn/ui + Tailwind | 快速开发，组件丰富 |
| **AI SDK** | Vercel AI SDK | 流式输出、function calling |
| **LLM** | 用户自选 (OpenAI/Claude/本地模型) | 灵活，不绑定 |
| **Agent 框架** | LangGraph / 自建编排 | 多 Agent 协调 |
| **向量库** | Supabase pgvector | 免费额度够用 |
| **数据库** | PostgreSQL (Supabase) | 存储学员、题目、进度 |
| **部署** | Vercel | Next.js 原生支持 |

## 6. LLM 灵活配置

### 用户自选 LLM 设计

```typescript
// 用户配置
interface LLMConfig {
  provider: 'openai' | 'anthropic' | 'local' | 'custom';
  apiKey: string;
  model: string;
  baseUrl?: string;  // 自定义 API 端点
}

// 示例配置
const configs: Record<string, LLMConfig> = {
  // 不同 Agent 可以用不同模型
  router: { provider: 'openai', apiKey: '...', model: 'gpt-4o-mini' },
  tutor: { provider: 'anthropic', apiKey: '...', model: 'claude-3-5-sonnet' },
  examiner: { provider: 'openai', apiKey: '...', model: 'gpt-4o' },
};
```

### 支持的 LLM 提供商

| 提供商 | 推荐模型 | 适用场景 |
|--------|---------|---------|
| OpenAI | GPT-4o, GPT-4o-mini | 通用，性价比高 |
| Anthropic | Claude 3.5 Sonnet | 代码理解强 |
| 本地部署 | Ollama + Llama3 | 隐私敏感，免费 |
| 其他 | 任何 OpenAI 兼容 API | 灵活扩展 |

## 10. 风险与边界

### 已知限制
1. **LLM 幻觉**：可能给出错误的 OS 知识 → 通过 RAG 引用来源缓解
2. **出题质量**：AI 生成的题目可能有歧义 → 人工审核 + 用户反馈
3. **上下文长度**：长对话可能超出限制 → 摘要 + 滑动窗口
4. **成本**：多 Agent 调用成本较高 → 用户自选模型控制成本

### 不做的事
- 不帮学员写代码（引导式教学）
- 不自动批改编程题（需要人工）
- 不替代训练营官方系统（只做辅助）
