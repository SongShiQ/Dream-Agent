# 进程与线程管理

## 进程的概念

**进程（Process）**是程序的一次执行实例，是操作系统进行资源分配和调度的基本单位。

### 进程与程序的区别

| 特性 | 程序 | 进程 |
|------|------|------|
| 本质 | 静态的代码文件 | 动态的执行过程 |
| 存储 | 磁盘上 | 内存中 |
| 生命周期 | 永久存在 | 有创建和销毁 |
| 资源 | 不占用资源 | 占用 CPU、内存等资源 |

### 进程控制块（PCB）

操作系统通过 **进程控制块（Process Control Block, PCB）** 来管理进程，包含：
- **进程标识符（PID）**：唯一标识进程
- **程序计数器（PC）**：下一条要执行的指令地址
- **寄存器值**：保存的 CPU 寄存器状态
- **内存管理信息**：页表、段表等
- **I/O 状态信息**：打开的文件、分配的设备
- **调度信息**：优先级、状态

### 进程状态

```
新建 → 就绪 ↔ 运行 → 终止
         ↑       ↓
         └── 阻塞 ┘
```

- **新建（New）**：进程正在创建
- **就绪（Ready）**：等待 CPU 调度
- **运行（Running）**：正在 CPU 上执行
- **阻塞（Blocked）**：等待某个事件（如 I/O）
- **终止（Terminated）**：执行完成

## 线程

**线程（Thread）**是 CPU 调度的基本单位，一个进程可以包含多个线程。

### 线程与进程的区别

| 特性 | 进程 | 线程 |
|------|------|------|
| 调度单位 | 资源分配单位 | CPU 调度单位 |
| 地址空间 | 独立 | 共享进程的地址空间 |
| 开销 | 创建/切换开销大 | 创建/切换开销小 |
| 通信 | 需要 IPC 机制 | 可直接访问共享内存 |

### 线程模型

1. **用户级线程**：由用户空间的线程库管理，内核不感知
2. **内核级线程**：由内核管理，创建/切换需要系统调用
3. **混合模型**：用户级线程映射到内核级线程

## 进程调度

### 调度算法

1. **先来先服务（FCFS）**：按到达顺序执行
2. **最短作业优先（SJF）**：执行时间短的优先
3. **轮转调度（RR）**：每个进程分配固定时间片
4. **优先级调度**：根据优先级选择
5. **多级反馈队列**：结合多种策略

### 调度指标

- **周转时间**：从提交到完成的时间
- **等待时间**：在就绪队列中等待的时间
- **响应时间**：从提交到首次响应的时间
- **吞吐量**：单位时间内完成的进程数

## 进程间通信（IPC）

1. **管道（Pipe）**：单向数据流
2. **消息队列**：结构化消息传递
3. **共享内存**：多个进程访问同一内存区域
4. **信号量**：用于同步
5. **套接字（Socket）**：网络通信
